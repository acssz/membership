Authentik 方面不存储用户的学历职位信息了（degree），需要在该平台上处理：
用户在 authentik 认证身份后，回到该平台上，需要自己声明自己的职业或正在读取的学历
如果是业界职员、学术雇员（RA、PostDoc）、或其他，无须进一步验证；
如果是学生，需要声明自己正在攻读的学历（本科、硕士、博士、非高等教育），并上传学生证照片。
可选的，如果能 OCR 瑞士学生证并且读取出信息就再好不过了。
学生每年（可配置）需要验证一次身份，没有验证身份的，在学历、职位栏明确标示未验证通过。
二维码作为 UI 上的主要部分，布局采用二维码居中，未认证和已认证通过 UI 的设计可以被明显辨认，身份信息放在二维码之下，除了职业和学历以外其他信息默认不展示（星标），有个小眼睛按钮可以点击切换展示状态，将“认证”功能做成单独的按钮/页面，而不是用一个 block 在主页上做。
